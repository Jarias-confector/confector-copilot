# User Experience (UX)

## Filosofía de la Experiencia

CONFECTOR Copilot no es un gestor documental.

No es un ChatGPT.

No es un ERP.

Es un asistente operativo.

El usuario nunca interactúa con tecnología.

Interactúa con su proyecto.

La IA debe desaparecer de la experiencia.

El usuario únicamente debe sentir que trabaja más rápido.

---

# Nuestro Usuario

El usuario principal es un Gerente de Proyecto.

Su día normalmente consiste en:

- Revisar avances.
- Resolver problemas.
- Coordinar contratistas.
- Atender clientes.
- Revisar costos.
- Dar seguimiento.
- Asistir a reuniones.
- Documentar acuerdos.

Nunca dispone de tiempo suficiente.

Generalmente trabaja desde:

- Laptop
- Celular
- Obra
- Oficina
- Automóvil

Muchas veces captura información por voz.

---

# El Problema

Los gerentes no quieren hacer documentos.

Quieren terminar tareas.

Ejemplos:

"No quiero hacer una minuta."

Quieren decir:

"Necesito enviar la minuta."

---

"No quiero hacer una presentación."

Quieren decir:

"Tengo una reunión con el cliente."

---

"No quiero buscar un acuerdo."

Quieren decir:

"Necesito responder una pregunta."

---

La plataforma debe entender la intención.

No únicamente la acción.

---

# Filosofía de Navegación

Nunca navegamos por funciones.

Siempre navegamos por objetivos.

Incorrecto:

Documentos

↓

Minutas

↓

Crear

↓

↓

Generar

Correcto:

Proyecto

↓

¿Qué deseas lograr?

↓

La plataforma realiza todo el proceso.

# La Pantalla Principal

Cuando el usuario abre el sistema no debe sentirse perdido.

Debe sentirse acompañado.

Ejemplo:

Buenos días Alejandro.

Proyecto actual:

Casa Vida.

Última actividad:

Hace 2 horas.

Se detectaron:

• 2 nuevos acuerdos.

• 1 riesgo.

• 5 pendientes.

¿Qué deseas hacer hoy?

[ Subir información ]

[ Preguntar al proyecto ]

[ Automatizar proceso ]

[ Abrir Workspace ]

Nada más.

La pantalla principal nunca mostrará:

50 botones.

20 módulos.

---

# Proyecto

Cada proyecto tendrá su propio espacio.

No habrá carpetas.

Habrá conocimiento.

Ejemplo:

Proyecto

Casa Vida

Estado General

Documentos

Audios

Conversaciones

Cronología

Dashboard

Copilot

Todo relacionado entre sí.

---

# Captura de Información

Subir información debe ser extremadamente simple.

Opciones:

Arrastrar archivos.

Grabar audio.

Pegar texto.

Importar carpeta.

Importar correo.

Nada más.

---

# Después de subir archivos

Nunca preguntar:

¿Qué deseas hacer con este PDF?

El sistema debe analizar automáticamente.

Mostrar algo como:

Encontré:

12 acuerdos.

7 pendientes.

2 riesgos.

4 responsables.

¿Deseas hacer algo con esta información?

---

# Copilot

El Copilot debe sentirse como un compañero.

No como un chatbot.

Ejemplo.

Usuario:

¿Quién quedó responsable del sistema hidráulico?

Respuesta:

Carlos López.

Compromiso adquirido en la reunión del 15 de junio.

Fecha compromiso:

20 de junio.

Fuente:

Minuta #18.

El usuario nunca debe preguntar dos veces.

---

# Automatizaciones

Las automatizaciones no son herramientas.

Son asistentes.

Ejemplos.

Preparar junta con cliente.

↓

Generar:

Resumen.

Pendientes.

Cronología.

Presentación.

Todo automáticamente.

---

# Workspace

El Workspace es el escritorio del gerente.

Aquí puede:

Editar.

Comparar.

Aceptar cambios.

Versionar.

Comentar.

Exportar.

No es Word.

No es PowerPoint.

Es una experiencia propia.

---

# Exportación

Nunca preguntar:

¿Quieres un Word?

Preguntar:

¿Cómo lo compartirás?

Cliente.

Interno.

Dirección.

Junta.

Dependiendo del contexto se generan formatos distintos.

---

# Configuración

Debe ser invisible.

Solo existirán opciones realmente necesarias.

Ejemplo:

Idioma.

Modelo IA.

API.

Empresa.

Branding.

Nada más.

---

# Estados Vacíos

Cuando un proyecto está vacío.

No mostrar tablas.

Mostrar ayuda.

Ejemplo:

Comienza agregando un audio o un documento.

Nosotros organizaremos el resto.

---

# Errores

Nunca decir:

Error de IA.

Error API.

Token inválido.

Traducir siempre.

Ejemplo:

No pudimos analizar este documento.

Intenta nuevamente o selecciona otro modelo.

---

# Feedback

Toda automatización debe indicar progreso.

Ejemplo.

Analizando audio...

Extrayendo acuerdos...

Identificando responsables...

Generando presentación...

Aplicando branding...

Listo.

El usuario debe confiar en el sistema.

---

Orden.

Confianza.

# La Regla de los Tres Clics

Ninguna tarea importante deberá requerir más de tres decisiones.

Ejemplo.

Proyecto.

↓

Automatización.

↓

Generar.

Fin.

---

# El Estado Ideal

Después de utilizar CONFECTOR Copilot durante varios días el usuario debe sentir que:

No busca documentos.

No redacta reportes.

No hace presentaciones.

No pierde acuerdos.

No recuerda dónde quedó algo.

Simplemente trabaja.

Todo lo demás lo hace la plataforma.

---

# Nuestra Mayor Diferencia

La competencia ofrece herramientas.

Nosotros ofrecemos tranquilidad.

El usuario debe sentir que su proyecto está organizado incluso cuando él no tiene tiempo para hacerlo.

CONFECTOR Copilot debe convertirse en el segundo cerebro del gerente de proyecto.
